# Payment System for Ads and Donations
*(As of February 25, 2026 – Consolidated from discussions)*

## Overview & Philosophy
- **Unified flow** for donations and ads: Core is the same Lightning Network direct-payment mechanism. Ads are "donation + ad slot premium" (extra sats for exposure on interstitials/shortlink pages).
- **Non-custodial end-to-end**: Every payment uses a Lightning invoice generated by the system → paid directly from sender (donor/advertiser) to recipient wallet. Read-only API access to recipient wallets for instant confirmation/on-chain verification. No funds ever touch site-controlled wallets → zero custody risk, no middleman fees/overhead.
- **Key differentiator**: Transparent, peer-to-peer funding for real projects (e.g., food forests in Kenya, desert farms in Namibia). No legacy charity opacity, no stock photos/pity narratives — real photos, updates, on-chain proof.
- **Ecosystem sites**:
  - 4t2.co: Shortlinks with impact interstitials (ads).
  - clickforcharity.net / directsponsor.net: Project details, verification, explanations.
  - roflfaucet.com / others: Gamification/PTC → coin earning → indirect influence on funding.

## Sender Flow (Donation or Ad)
1. Sender lands on donation/ad form (e.g., via shortlink interstitial, direct donation page, or ad purchase).
2. **Dropdown selection**:
   - List of active/verified projects (pulled from directsponsor.net data).
   - Final option: **"Auto-allocate"** (supports fair distribution across projects + rewards community effort via gamification).
3. Sender chooses slot/position/duration (for ads only → determines premium amount).
4. System generates Lightning invoice:
   - Amount: Base donation (if pure) or donation + ad premium.
   - Destination: Chosen project's wallet (direct) OR current "best" project's wallet (if auto).
5. Sender pays → read-only API confirms instantly → sats credited to recipient wallet (on-chain verifiable).
6. Post-payment: Show tx proof, project name, brief impact note.

## Auto-Allocate Logic ("Lazy" / Default Option)
- Routes to the "best" project based on pre-set, transparent criteria (configurable in backend/admin).
- Example criteria (pick one or hybrid):
  - Project with **lowest sats received** in last 30 days (spreads funding evenly).
  - Highest user/community influence score (from prior gamification/PTC allocations).
  - Closest to a posted funding milestone (if projects set goals).
  - Round-robin rotation among eligible projects.
- Rules published openly (e.g., on 4t2.co/impact): "Auto-allocate sends to the project with fewest recent sats, evaluated at time of payment."
- Still direct invoice → no pooling/holding.

## Monthly Coin Valuation & Reward (Gamification Tie-In)
- **Pool composition** (general support revenue):
  - All ad revenue (interstitials/shortlinks).
  - All auto-allocated donations (lazy choices).
  - **Exclude**: Specifically directed donations (sender picked a project → 100% to that wallet, no dilution).
- **Calculation** (end of each month):
  - Total pool (in sats, or USD equiv at receipt time if mixed).
  - Value per coin = pool ÷ total active coins held/earned by users.
- **Outcome**:
  - "Spend"/deduct coins used in calc (they've "cashed out" their value).
  - Preserve ongoing/long-term points balance (leaderboards, badges, perks).
- **User benefit**: Coins retroactively represent effort/contribution → lazy auto-donations/ad revenue indirectly reward grinders (tasks, PTC, sharing).
- **Messaging**: "Your coins helped direct X sats this month from general support! Double karma for lazy donors — they fund projects fairly + support the community."

## Implementation Notes
- **Code reuse**: Same invoice gen + read-only monitoring for ads/donations. Ads just add amount multiplier based on slots.
- **Transparency tools**:
  - Dashboard: Show month-end summaries, tx proofs, "how your auto-donation routed."
  - Explanation pages: Diagram money flow, live projects, criteria rules.
- **Edge cases**:
  - No active projects? Fallback to a default (e.g., balanced split or hold for next month — but prefer always routing).
  - Volatility: Use receipt-time rates if converting; focus on sats for purity.
- **Scalability**: Easy to add projects; auto criteria keep it fair without manual intervention.

This system stays radically honest: direct, verifiable, motivating for users, zero custody burdens. All payments Lightning-native, impact visible on-chain and via project updates.
